Locutus 4.0 ships the full TypeScript migration tracked in [Discussion #532](https://github.com/locutusjs/locutus/discussions/532) and [PR #535](https://github.com/locutusjs/locutus/pull/535).

## What changed

- `src/` is now fully TypeScript (`553` `.ts` files, `0` `.js/.mjs` files)
- Exports moved from default to named exports
- Minimum Node version is now `>=22`
- Published `dist` remains CommonJS-compatible for deep `require(...)` with named access
- Function pages now include first-class code variants (`Module`/`Standalone`, TS/JS) and copy buttons

## Breaking changes

### Named exports (default imports break)

Before:

```ts
import sort from 'locutus/php/array/sort'
```

Now:

```ts
import { sort } from 'locutus/php/array/sort'
```

### Node engine bump

```json
"engines": {
  "node": ">= 22"
}
```

### `locutus/php` root namespace changes

`locutus/php/index` no longer exports `var` and `net-gopher` as root namespace properties.

Reason: ESM namespace export keys must be valid identifiers; `var` is reserved and `net-gopher` has a hyphen.

Use deep imports:

```ts
import * as phpVar from 'locutus/php/var'
import * as phpNetGopher from 'locutus/php/net-gopher'
```

## Compatibility (what still works)

Valid:

```js
const { sort } = require('locutus/php/array/sort')
```

Invalid in v4:

```ts
import sort from 'locutus/php/array/sort'
```

```js
const sort = require('locutus/php/array/sort')
const { var: phpVar } = require('locutus/php')
const phpNetGopher = require('locutus/php')['net-gopher']
```

## Upgrade

1. Upgrade package:

```bash
npm install locutus@^4
# or
yarn add locutus@^4
```

2. Ensure Node `>=22` where Locutus runs.
3. Replace default imports with named imports.
4. Replace `locutus/php` root access for `var` and `net-gopher` with deep imports.
5. Run your typecheck/tests.

## Why this release matters

A major driver from [#532](https://github.com/locutusjs/locutus/discussions/532) was type-definition drift: `@types/locutus` and consumer-local types were hard to keep in sync. In v4, runtime code and types now ship from one source.

For contributors, this release also adds stronger CI type-quality gates (rejecting `@ts-nocheck`, `@ts-ignore`, and API/type-regression patterns) and keeps JS/MJS additions on an explicit allowlist.
