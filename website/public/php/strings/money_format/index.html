PHP's money_format in JavaScript

Here’s what our current JavaScript equivalent to PHP's money_format PHP's money_format looks like.

[ View on GitHub | Edit on GitHub | Source on GitHub ]

How to use

npm is the recommended installation method. It pairs nicely with a CommonJS module bundler such as Browserify, webpack or rollup.js.

You you can install via npm install --save --save-exact locutus and require it like so: var money_format = require('locutus/php/strings/money_format') and start using money_format. You could also require the strings module in full.

ES5/ES6

This function targets ES5, but as of Locutus 2.0.0 we also support ES6 functions. Locutus transpiles to ES5 before publishing to npm.

A community initiative

Locutus offers community built functions and goes by the McDonald’s Theory. We’ll put online functions that aren't perfect, in the hopes to spark better contributions. Do you have one? You can directly edit this file on GitHub, this will create a Pull Request that the core team can review, and merge into our codebase.

This way of working also means that we don't offer any production guarantees, and recommend to use Locutus inspiration and learning purposes only.

Notes

  • This depends on setlocale having the appropriate
    locale (these examples use ‘en_US’)

Examples

Please note that these examples are distilled from test cases that automatically verify our functions still work correctly. This could explain some quirky ones.
